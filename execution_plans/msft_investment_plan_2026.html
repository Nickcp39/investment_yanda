2 为什么是现在(买入依据)

  • 价格首次进入 STARTER 区间:$384 已低于 base 8% 公允价 $436,MOS +13%;当前对应 10 年 IRR ~9.4%(高于 8% 门槛)。此前 $555 是"等",现在"在区内、可以开始"。
  • 生意本身没坏:Q3 FY26 Azure +40%(重新加速)、Microsoft Cloud $54.5B(+29%)、Commercial RPO $627B(+99% 已签约)、AI run-rate $37B(+123%)——需求侧一手确认,不是 promise-ware。
  • 资产负债表极硬:净现金 ~$47B、债务下降、OCF $170B TTM、连续 20+ 年增派 dividend、45.6% 营业利润率。bear 案的"活下来"是可数学证明的。
  • 市场情绪已冰点(可能已过头):FY26 capex 指引 ~$190B(+61%)引发恐慌,股价 −32%。但 pipeline 拆解显示瓶颈在产能不在需求——RPO +99% 大于交付能力,capex 是追需求不是赌需求。这是"恐慌型 de-rate"不是"基本面恶化"。
  • 但 Q4 FY26 财报 3 周内落地(~2026-07 下旬)——FY26 全年 capex 是否坐实 $190B、是否首次拆维护/成长、OpenAI 口径、Azure FY27 指引,可能改变故事。→ 所以不梭哈、财报前只建 50%。
结论:生意 exceptional + 价格有 MOS + 但财报是关键变量 → 两段式定投:财报前 50%、财报后 50%,让节奏吃掉时机不确定性。

4 持有规则(建完之后)

  • 建完即转长期核心仓,任何回撤都不动。这笔在 STARTER 区间买的,就是为了骑 5–10 年 owner-earnings 复利(pipeline base 案 IRR ~9.4%、bull 15%、bear breakeven)。
  • 不看短期价格做决定。建仓期只跟定投表;建完只做两件事——(a) 拿住 (b) 每季读一次财报和 pipeline monitor.md,KPI 盯 K-A 到 K-E。
  • 牛市途中不追高加仓(超 $436 no-chase 就是纪律,不是建议);不因短期暴涨改变总额度。
  • 补完 O2/O3 缺口 + 完整度上 80% 是唯一"评 CORE"的路径 —— 那是重开 panel、重签合同,不是当场加仓。

5 加仓与卖出(指针,不在本文件即时执行)

  • 价格加仓:≤ $303 触发尾弹(见 §3);同时补 O2/O3 → 重开 panel 评 CORE(目标 15%+),不在此文件决定。
  • 价格停买:≥ $436(隐含 IRR < 8%),丧失 MOS → 停买、持有已建仓位。

7 纪律机制(全文的心脏)

  • 时间代替决定——到日期就买,永不临场问"该不该"。财报当天特别不做决定。
  • 永远分批——财报好/坏/中性每种结局我都"部分对",没有一种能让我恨自己。
  • "再等等更便宜"是要管住的对象——每个 STARTER 区都有这个念头。它既是恐慌,也是"我还没准备好签合同"。感受它,但别让它接管定投表。
  • 财报后不追涨也不砸盘——三态判断只走 §3 分支,不即兴。
  • 偏离要付成本——任何想改计划的念头:强制 72 小时冷静 + 书面写理由(贴到 §8 违约记录)。
  • 一次买贵不致命,破坏纪律才致命。MSFT base 案 9.4% 年化就跑不赢 QQQ 19% 基准——只有纪律 + 生意质量 + 下行保护合起来才是这单能做的理由。任何一环松了,单持 MSFT 就没有超额意义。
