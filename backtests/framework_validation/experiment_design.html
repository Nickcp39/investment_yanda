一眼看全流程

旧 10 层并没有删除，只是被压进 6 个执行模块

旧 10 层适合做能力检查，确保没有漏看方向、生意、证据、财务、产业链、护城河、管理层、反演、估值和决策。新 6 模块适合实际跑 case，因为每个模块都必须输出 signal，最后能落到仓位。

旧 10 层新执行位置为什么这样合并
1. Direction SelectionM2 Theme / Mechanism方向不单独成文，直接进入“为什么现在、市场重估什么、瓶颈是什么”。
2. Business Thesis / Quality GateM2 Theme / Mechanism第一性原理和一页好生意承保是 thesis 的核心，不再另开文件。
3. Evidence Engineering01 Source Pack + M1 Evidence Spine原始材料和事实可信度分开，M1 只控制 confidence/haircut。
4. Accounting Trend ValidationM4 Financial Reality财报必须回答机制是否被验证，而不是机械看利润表。
5. Industry / Value ChainM3 Profit Pool / Durability产业链只有在说明“谁拿走利润池”时才有决策意义。
6. Moat / BottleneckM3 Profit Pool / Durability护城河和瓶颈合并判断：短期稀缺是否能变成持久经济性。
7. Operator UnderwritingM3 Profit Pool / Durability管理层不单独漂浮，放进“利润能否被正确配置和保住”。
8. Inversion / RiskM5 Inversion / Trap Test反演升级成专门的 veto 模块，负责抓假便宜和永久损失。
9. Valuation / Margin of SafetyM6 Price / Position / Monitor估值直接转成 buy-below、starter/add/trim 区间。
10. Decision / MonitoringM6 + QA/Score决策卡锁定仓位，后续 audit/score/tune 负责复盘学习。

评分与调权逻辑

M1 不是 veto

证据完整度只影响仓位 haircut 和监控强度，避免把早期赢家压成 WATCH / 0%。

M2/M3/M4 形成 conviction

主题机制、利润池、财务现实一起决定是否值得拥有。

M5 可以硬 veto

结构性衰退、假便宜、资产负债表死亡、永久损失可以把 size 设为 0。

M6 输出仓位

价格不是一句贵/便宜，而是 buy-below、starter、add、trim、exit。

聚合模型

conviction_raw = W[context].M2 * signal[M2]
               + W[context].M3 * signal[M3]
               + W[context].M4 * signal[M4]

risk_penalty = W[context].M5 * abs(min(signal[M5], 0))

if hard_veto(context, M5, M4):
  target_size = 0
else:
  conviction = conviction_raw - risk_penalty
  target_size = size_function(context, conviction, M6.price, M1.confidence)

Context 权重方向

Context主驱动价格角色主要防错
quality_compounderM2, M3, M4调节仓位，不自动 veto不要因为经典倍数偏高就卖掉好生意。
exceptional_bottleneckM2, M3, M4结构化入场，控制下行不要因为 PE/MOS 不舒服就错过小仓位 starter。
cyclical_inflectionM2, M4, M5用正常化周期盈利不要把 trough 或 peak earnings 当永久现实。
structural_decline_trapM3, M4, M5veto 后价格基本无关低 P/E 不是安全边际。
false_cheap_value_trapM4, M5先正常化盈利再谈便宜防 PFE/PYPL/INTC 式假便宜。
