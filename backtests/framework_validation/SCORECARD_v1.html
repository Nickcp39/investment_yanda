两个系统性发现(≥3 案例,过防过拟合门槛)

① 天花板低一档 —— "起步对、封顶太怂"

4 个赢家(GOOGL · NVDA · META · AAPL)全是 size-dist 1、too_small_missed_asymmetry。STARTER 入场对(3–4%),但 durability 天花板(9–12%)把本该进 Core(15–25%)的复利股压在了 Confirmed。扣分点是顶,不是触发。

→ 含义补丁(待 PATCH_LEDGER 起草验证):当 操作者分高 + 耐久性高 + 无否决 时,放行到 Core 天花板。"在天花板上更勇,而不是在扳机上"。

② 升级缺口 —— 不可预见催化剂的困境反转(唯一 FAIL)

INTC-2024:可生存的困境名,结局靠 as-of 之后的外部救援(换帅 + ~$16B 政府/英伟达/软银注资)。框架机械归零(WATCH/0%/TRIM)。反事实证明:把催化剂喂进去,框架不改任何权重就翻成 STARTER —— 是"预见"失败、不是"加权"失败。

→ 含义补丁:可生存 + 催化剂依赖的反转,应标红升级给人 + 做催化剂监控(融资缺口=触发线),而不是机械归零;再加一条小护栏防止 survivable self-funded reset 被自动 TRIM。这正是先前讨论的 assumption_register / 人工升级机制的活证据。

什么已经成立(别改坏)

  • 陷阱精度 4/4:M4/M5 陷阱机器正确避开 PFE/PYPL/DIS/INTC-2021。改天花板时绝不能破坏这个——框架谨慎的原因和它避雷的原因是同一个。
  • "完整度=haircut 非否决"修复生效:除了那个可预见性 case,赢家无一被 WATCH/0% 封顶。
  • 操作者分→天花板,双向都对:高分抬顶(GOOGL/META 4–3/5),低分压顶(DIS 2.5/5)——PATCH_LEDGER P1 已被 GOOGL+META+DIS 三案验证。
  • 原始 GOOGL 病已治:同一锚点,旧框架 WATCH/0% → 新框架 STARTER。
