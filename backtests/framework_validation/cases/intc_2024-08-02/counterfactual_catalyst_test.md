## 4. Failure-type interpretation (the whole point)

The counterfactual verdict **FLIPS to a real position (STARTER, 3%, HOLD-an-existing-holder)**. Per the
test's own decision rule:

> *If the verdict now FLIPS to a real position (STARTER+): the blind failure was **FORESIGHT only** —
> the framework CAN weigh the catalyst when known.*

### ⇒ This is a FORESIGHT-only failure, NOT a WEIGHING flaw.

**Evidence that it is foresight, not weighing:**

1. **When handed the catalysts, the machinery moves correctly and proportionately.** The same six
   modules, the same aggregation model, the same context gates and §5 STARTER test produce a STARTER —
   without any change to weights or gates. The framework's *weighing apparatus is not broken*: feed it
   the resolved binding constraint and it clears the veto and sizes in. A genuine weighing flaw would
   have kept the trap-veto dominant *even with* a funded, well-led, sovereign-backed turnaround — and it
   does not.
2. **The catalysts map one-to-one onto the blind card's own named binding constraint and kill-criteria.**
   The blind Runner did not mis-identify the pivot — it correctly isolated "the foundry
   cash-burn-vs-thin-OCF gap" as *the* binding constraint and wrote kill-criteria #2/#5/#6 around exactly
   the things (cash coverage, 18A, leadership/strategic backing) that the catalysts resolve. The
   framework knew *what to watch*; it simply could not *know* that those watched items would resolve
   favorably. That is a foresight limit, by definition.
3. **The blind run's own `runner_dissent` already flagged the two-sided edge.** The locked card states:
   "a disciplined value investor could argue for a tiny TRACKING sliver … there IS a clean, identifiable
   key variable (foundry trough + cash coverage)," and explicitly named the "too-harsh edge." The Runner
   *sensed* the sensitivity. What it did **not** do — and what the test says it *should* have done — is
   escalate that sensitivity into the verdict structure itself.

### The precise nature of the blind miss

The blind Runner's error was **not** "it weighed the catalyst wrong" (it never had the catalyst). The
error was that it **mechanically resolved a genuinely catalyst-sensitive, survivable, sub-book reset all
the way to 0% / TRIM / "bad"** instead of **flagging the verdict as highly sensitive to an unknowable
turnaround catalyst.** A name where:

- near-term survival is **not** in question (the card's own M5 concedes this), **and**
- there is a **clean, named, binary key variable** (foundry trough + cash coverage + leadership), **and**
- the downside, while real, is **not** imminent-insolvency,

…is the textbook profile of a **"tracking-to-STARTER, catalyst-monitor"** case, **not** an
avoid/trim. The discipline failure is collapsing a two-sided, monitorable bet into a one-sided avoid —
*especially* after the dissent had already identified the fork.

### Implication (one line)

**This class of case needs human escalation + a catalyst-monitor flag — not a mechanical-weighting
patch:** the blind Runner should have explicitly tagged the verdict as *highly sensitive to an
unknowable turnaround catalyst* (new operator / external-funding / 18A) and parked the name as a
**tracking-to-STARTER catalyst watch with the funding gap as the single trip-wire**, rather than zeroing
it to WATCH/0%/TRIM with avoid framing. The framework can weigh the catalyst once it exists; the gap is
in **surfacing catalyst-sensitivity at decision time**, not in the arithmetic.

> **Caveat (kept honest):** "FORESIGHT-only" is *not* a free pass. A disciplined blind process could
> still have been one notch better **without any lookahead** — by letting its own survivable-balance-
> sheet + named-key-variable findings pull it to a small tracking-to-STARTER posture (per
> WEIGHTING_HARNESS §0's stated goal of "treating uncertainty as a size-haircut, not a veto"), rather
> than an avoid/trim. So the practical fix is two-part: (a) a **catalyst-sensitivity escalation flag**
> (the main fix), and (b) **tightening the §6 trap-veto so a *survivable, sub-book, self-funded* reset
> with a clean key variable is not allowed to be mechanically zeroed** — a guard-rail, not a full
> re-weight. The dominant lesson remains: **escalate the foresight-sensitivity; don't bolt on a
> mechanical patch that would start buying every cheap falling knife.**
