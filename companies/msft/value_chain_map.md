# MSFT Value Chain Map (Stage 4C · M3)

最后更新: 2026-06-19

## 价值链定位
微软横跨**企业 IT 价值链的多层**,且在 AI 时代向上(模型)和向下(芯片/数据中心)双向延伸:

```
半导体 (NVDA GPU / 自研 Maia) → 数据中心/电力 (capex $190B 量级) →
  云基础设施 (Azure) → 平台/中间件 (AI Foundry, 数据, 身份 Entra) →
    应用 (M365 / Dynamics / GitHub) → AI 助理层 (Copilot) → 企业终端用户
```

## 利润池捕获
- **最肥的两段**:应用层(M365,~高利润订阅)+ 云基础设施层(Azure,规模经济 + 消费计费)。微软同时占两段是其与纯云(AWS 偏基础设施)/纯应用厂商的结构性优势。
- **AI 层**:Copilot(应用内变现)+ Azure OpenAI/Foundry(算力 + 模型 API 变现),AI run-rate $37B 是这层的早期捕获。
- **议价权**:对上游(GPU/电力)是被动接受者(capex 涨价 ~$25B 部分来自组件涨价,facts O1);对下游企业客户是强议价方(捆绑 + 迁移成本)。

## 关键依赖与风险
- **上游集中**:GPU 供给(NVDA)+ 电力/数据中心产能是瓶颈,推高 capex。
- **OpenAI 关系**:既是 Azure 增长引擎又是依赖项;经济学拆分未一手验证(**OPEN O3,重大**)。
- M3 价值链信号:**+1**(占据最肥两段 + AI 双向延伸,但上游受制于 GPU/电力 + OpenAI 依赖待澄清)。
