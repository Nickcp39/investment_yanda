# META Inversion / Trap Test (M5) — as_of 2026-06-19

> 反过来想：什么会让在 $577 买入 META 变成永久减值或大错？

## 失败路径
| # | 失败路径 | 机制 | 是否结构性破裂 | 当前读数 |
|---|---|---|---|---|
| F1 | **capex 黑洞坐实** | FY26 $125–145B capex 持续上调但营收/营业利润不随之复利 → ROIC 跌破资本成本，owner earnings 被永久吃掉 | 准结构性（可降耐久性，非资产负债表死亡）| capex 翻 3.5x，ROI 无一手量化（O4）；Q1 单价 +12% 与"capex 有用"方向一致但未证 |

| F3 | **OS/隐私层冲击重演（ATT 2.0）** | Apple/Google 再砍归因数据 → 广告 ROI 与单价回落 | 非结构性（一次性 reset，曾恢复）| 无新冲击；结构性依赖在 |

## 永久减值 vs 可恢复
- **资产负债表死亡风险 = 无**：净现金 ~$22.4B、OCF $32B/季强劲 → 即便 bear 兑现，资产负债表能存活，bear ~−42% 是**de-rate + 减值，非破产**。
- **唯一可升 veto 的路径 = F2**（流量入口被 AI 结构性改写，广告引擎耐久性破裂）。当前无证据 → **不触发硬 veto**。
- F1（capex 黑洞）是**最可能成真的价值毁灭路径**，但它降的是 owner earnings 与耐久性（→ haircut + size 控制 + kill 监控），不是资产负债表存活 → 按修好的规则**不是 veto，是 size 控制**。

## 假便宜陷阱测试
- META 现价 **不是"假便宜"**——它**不便宜**：base IRR +4% < 8% 门槛，FCF yield 仅 3.4%。这是"好生意、价格已充分到偏贵、owner-earnings 锚未决"的情形，不是"低估值陷阱"。
- 真正的陷阱是**反向的**：被"广告满血 +33%/+12% 单价"和 AI 叙事带跑，忽视 FCF 被 capex 吃光、base IRR 不过门槛。

## kill 标准（搬入 monitor）
- K1: 营收增速跌到个位数 **且** 单价转负（广告引擎质量证伪）。
- K2: capex 继续上调但营业利润不增（F1 capex 黑洞坐实）→ ROIC 跌破资本成本。
- K3: DAP 连续下滑 + 时长下降（F2 流量入口改写早期信号）。
- K4: 新的 OS/隐私或监管冲击实质削广告归因（F3/F4）。
- K5: 买入价隐含 base IRR < 门槛（现价即接近触发，故价格本身封顶）。

## M5 signal
**−1（警告，非 veto）**: 失败路径真实（F1 capex 黑洞最可能、F2 唯一可升 veto 但无当前证据），但无结构性破裂、无资产负债表死亡。前瞻不确定性（capex ROI + 价格）= haircut + size 控制，不是硬 veto。
