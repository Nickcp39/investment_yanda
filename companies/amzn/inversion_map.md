# AMZN Inversion Map (M5) — as_of 2026-06-19

> 反过来想: 什么会让 AMZN 这笔投资永久减值？哪些是 size 控制 / haircut，哪些可升 veto？
> 修好的规则: **硬 veto 仅在 thesis 耐久性结构性破裂 或 资产负债表无法存活时触发**；前瞻不确定性（capex ROI、AWS run-rate）= haircut，不是 veto。

## 失败路径
| # | 路径 | 是否永久减值 | 级别 |
|---|---|---|---|
| F1 | **$200B/年 AI capex ROIC < 资本成本** → 折旧浪潮 + 利息侵蚀 owner earnings，价值毁灭 | 是（价值毁灭，非破产）| **size 控制 + 监控（最强单一风险）**；唯一可能升 veto 的路径但需多年证据 |
| F2 | AWS 份额被 Azure(OpenAI)/GCP + 客户自研结构性侵蚀，AI 工作负载"可移植"削弱转换成本 | 可能（利润池侵蚀）| haircut/监控；大规模份额坍塌才升 veto |
| F3 | 零售利润率回吐（创纪录 13.1% 不可持续 / 关税 / 竞争 / 宏观）| 部分（周期）| haircut |
| F4 | 加杠杆（长债 $119B、季增 $53B）遇利率上行 + 需求软化双杀 | 否（OCF $148.5B 可服务）| 监控 |
| F5 | GAAP 净利被 Anthropic 公允价值收益失真，反向波动时净利大跌（情绪/估值冲击）| 否（非现金、非经营）| 监控（估值口径项）|
| F6 | **估值: 现价无 MOS，base IRR 负** → 即便生意好，付太高价 = 永久低回报 | 是（机会成本永久损失）| **价格封顶 verdict（实际生效的卡点）** |

## 永久减值 vs 可恢复
- **可升 veto 的唯一路径 = F1（capex 价值毁灭）**，但需连续几年 ROIC 证据，当前无法证伪（O4）→ 现在是 haircut + size 控制 + 监控，不是 veto。
- **资产负债表不会死**: OCF $148.5B 巨大、净现金（金融债）仍正 ~+$24B → 不触发"资产负债表无法存活"的硬 veto。
- **F6（估值无 MOS）不是 veto 而是封顶**: 生意没破裂，是价格不对 → verdict 封在 WATCH（同 GOOGL），不是 REJECT。

## Kill 标准（→ monitor.md）
- K1: 连续多季 capex 大增但 AWS 增量营收/利润不匹配（capex ROIC 恶化信号）→ F1。
- K2: AWS 增速结构性回落到 <15% 且份额数据走弱 → F2。
- K3: 零售营业利润率从 13.1% 显著回吐（<9%）→ F3。
- K4: 净债务恶化 + OCF 增长停滞（资产负债表弹性耗尽）→ F4。
- K5: 价格进一步上行（base IRR 更负）→ 维持 no-chase；价格跌破 ~$175（base IRR 回 8%）→ 解封顶、可评 STARTER。

## 结论
- **无硬 veto**（thesis 未结构性破裂、资产负债表能存活）。
- **实际卡点 = F6 估值无 MOS**（价格封顶 WATCH），叠加 F1 capex ROI 前瞻不确定（haircut）。这是"好生意 + 错价格 + 重投资期 owner earnings 模糊"，与 GOOGL 同构、与 NVDA（价格友好）相反。
