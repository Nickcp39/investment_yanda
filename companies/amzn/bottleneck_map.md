# AMZN Bottleneck Map (M3) — as_of 2026-06-19

## AMZN 是不是"瓶颈"？
- **零售/物流**: AMZN 是西方电商基础设施的近-瓶颈（卖家必须在场），但有 Walmart/Shopify 等替代 → **强渠道，非独占瓶颈**。
- **AWS**: 云三巨头之一（Azure/GCP 并存）→ **寡头之一，非独占瓶颈**（与 NVDA 的 AI 算力独占瓶颈不同）。
- **AMZN 自身受制于的上游瓶颈**: ①AI 算力（NVDA GPU——AMZN 用自研 Trainium 部分绕过）；②电力/数据中心建设产能；③资本（$200B capex 需 OCF + 加杠杆）。

## 关键稀缺资源（AMZN 捕获的）
- 配送网络密度 + Prime 飞轮（零售）；企业上云的转换成本 + 数据引力（AWS）。
- **AMZN 不是"必经瓶颈"型生意（如 NVDA/CUDA），而是"多引擎规模复利"型生意**——这影响估值框架: 不能给瓶颈溢价，须按复利机器的正常化 owner earnings 估值。

## 当前的真瓶颈（约束 AMZN 增长的）
- **资本 + 电力 + 数据中心建设速度**: 管理层称"边装边变现、需求 > 供给"（S006，待核）→ 当前瓶颈是**供给侧（能建多快）**而非需求侧。这支持 AWS 增长可持续的多头看法，但也正是 $200B capex 的来源（双刃）。

## 结论
- AMZN 是**强渠道/寡头利润池**，非独占瓶颈。估值须按复利机器（正常化 owner earnings），不享瓶颈溢价——这与 valuation.md 的"价格无 MOS"一致。
